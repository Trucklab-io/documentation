---
title: "Invite your team"
description: "Create workspace access and assign an appropriate role to each team member."
icon: "user-plus"
---

Owners can create user access from Settings and choose whether each person
works as an Admin or Regular user.

## Before you begin

- You must have the **Owner** role.
- Confirm the person's email address and required role.
- Prepare an initial password of at least eight characters.

<Warning>
  The current access form creates credentials directly. Share an initial password
  through a secure channel and ask the user to replace it using the password-reset flow.
</Warning>

<Steps>
  <Step title="Open workspace access">
    Open **Settings**, select **Users**, and find **Access list**.
  </Step>
  <Step title="Enter the user details">
    In **Invite user**, enter the email address, first name, last name, and initial password.
  </Step>
  <Step title="Choose a role">
    Choose **Admin** for someone who manages workspace data and reviews planning.

    Choose **Regular** for someone who works with operational information and draft planning.
  </Step>
  <Step title="Create access">
    Select **Send invitation**. This creates the user's access with the temporary
    password you entered. Confirm that the user appears in **Access list**.
  </Step>
  <Step title="Verify the first sign-in">
    Ask the user to sign in and confirm they can see the expected workspace and pages.
  </Step>
</Steps>

## Choose the least access required

| Role | Use it for |
| --- | --- |
| **Owner** | Workspace ownership, user management, all settings, and planning review. |
| **Admin** | Workspace data, operational settings, and planning review. |
| **Regular** | Assigned operational information and draft planning work. |

See [Roles and permissions](/concepts/roles-and-permissions) for the full comparison.

## Remove access

An Owner can remove a user from **Access list**. Confirm the correct email before
deleting access because the operation affects that user's workspace session.

<Note>
  If an existing user cannot sign in, try password reset before deleting and
recreating the user.
</Note>

## Update several users

Owners can select multiple rows in **Access list** and use the bulk toolbar to:

- change the selected users to **Admin** or **Regular**;
- change access to **Active** or **Inactive**;
- delete the selected user access records.

Review the selection count before applying a bulk action. You cannot use the user
form to create another Owner.
