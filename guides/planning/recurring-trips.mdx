description: "Build, filter, pause, resume, and maintain daily, weekly, or monthly trip templates."

Recurring trips reduce repeated entry for stable routes. A template generates
planning occurrences when you open a matching client and date.

## Create a template

<Steps>
  <Step title="Open Recurring trips">
    Select **Recurring trips**, then **New recurring trip**.
  </Step>
  <Step title="Configure Basics">
    Select the client and an optional default driver. Choose **Active** to generate
    occurrences or **Paused** to save without generation.
  </Step>
  <Step title="Set the Schedule">
    Choose **Daily**, **Weekly**, or **Monthly**. Select at least one allowed weekday
    for daily or weekly work, or choose the monthly day. Add start and end dates.
  </Step>
  <Step title="Enter the Route">
    Add the departure time, optional duration, required addresses, load, distance,
    and optional manual price. Selecting a client can prefill its address as departure.
  </Step>
  <Step title="Review and create">
    Review every section, select **I confirm the information is correct**, and select **Create trip**.
  </Step>
</Steps>

## Choose the recurrence

| Frequency | Configuration |
| --- | --- |
| **Daily** | One or more allowed weekdays; all seven means every day. |
| **Weekly** | One or more operating weekdays. |
| **Monthly** | A numbered day of the month. |

Use date boundaries for seasonal or contractual work. An end date prevents future
generation after the operating period.

## Generate the occurrence

Open **Planning**, select the same client, and load a matching date. Affretio creates
the occurrence when that planning is opened.

<Note>
  The template and each generated trip are separate records. Editing or pausing the
  template does not rewrite trips that were already generated.
</Note>

## Find and maintain templates

Search by client, driver, route, or recurrence. Filter by client and **Active** or
**Paused** status. Use 15, 25, or 50 records per page.

- Edit a template when the stable future pattern changes.
- Pause a template to stop future generation without deleting it.
- Resume it when the same pattern should operate again.
- Use bulk status actions for a reviewed selection.
- Delete only after checking already generated planning.

## Avoid duplicates

1. Search the route and client before creating another template.
2. Use one template for one stable recurrence pattern.
3. Pause or edit the existing template rather than cloning it casually.
4. Open a matching planning date after the change.
5. Search the resulting trips before creating anything manually.

<Warning>
  Bulk pause, resume, or delete affects every selected template. Confirm the filtered
  list and selected count before applying it.
</Warning>
