Planning organizes one client's trips for a selected day. Affretio stores edits in
a numbered version so the team can distinguish draft work from approved operations.

## Before you begin

- Create the client.
- Add drivers and trucks you expect to assign.
- Record known driver absences.
- Confirm the active workspace and your role.

## Understand the Planning layout

| Area | Purpose |
| --- | --- |
| **Trajets** | Create, find, and work with trips. |
| Date header | Select **Jour du planning**, move to the previous or next day, and read the active version. |
| **Mes camions disponibles** | Review vehicles available for assignment. |
| Filter bar | Search drivers or trips and filter by status. |
| Schedule board | Review driver timelines and assigned work. |
| **Actions** | Submit, approve, reject, restore, print, or export when available. |

## Open the correct planning

<Steps>
  <Step title="Open Planning">
    Select **Planning** in the sidebar.
  </Step>
  <Step title="Choose a client">
    Choose the client organization. If the directory is empty, use **Ajouter un client**.
  </Step>
  <Step title="Choose the day">
    Use **Jour du planning** or the arrow buttons to open the operating day.
  </Step>
  <Step title="Confirm the version">
    Read the version badge. Make sure you are editing a Draft, not only reviewing
    an Approved or Submitted snapshot.
  </Step>
</Steps>

## Create a trip

<Steps>
  <Step title="Start the trip">
    Open **Trajets** and select **Créer un trajet**.
  </Step>
  <Step title="Confirm the day">
    Enter the trip day. The creation flow treats this as a required, fixed identity
    for the trip; create the trip on the correct date.
  </Step>
  <Step title="Assign resources or plan later">
    Choose a driver and compatible truck, or use **Assigner plus tard**. An absent
    driver cannot be assigned on a covered date.
  </Step>
  <Step title="Enter schedule information">
    Add departure and arrival times. If you enter an arrival, you must also enter
    a departure. Affretio represents overnight arrival with a day offset.
  </Step>
  <Step title="Enter the route">
    Add complete departure and arrival addresses. Select a matching address suggestion
    when one is available.
  </Step>
  <Step title="Add commercial and operating details">
    Enter load, distance, manual price, and status when known. A new trip starts as **Planifié**.
  </Step>
  <Step title="Save the trip">
    Save the form and verify that the trip appears in the current version.
  </Step>
</Steps>

## Required and conditional fields

| Field | Rule |
| --- | --- |
| Trip day | Required. |
| Departure address | Required. |
| Arrival address | Required. |
| Departure time | Required when an arrival time is entered. |
| Driver and truck | Optional when you choose **Assigner plus tard**. |
| Distance and price | Optional for planning, but may block pre-invoicing later. |

<Warning>
  Always verify the selected client, date, and version before adding work. A correct
  route in the wrong client planning is still an operational error.
</Warning>
